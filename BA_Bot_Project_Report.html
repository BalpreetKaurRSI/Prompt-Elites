1. Project Overview

BA Bot is an AI-powered SDLC Productivity Assistant designed to help Business Analysts, Product Managers, and Agile teams rapidly translate free-form requirements into structured, actionable backlogs. The tool leverages Anthropic's Claude AI to intelligently decompose requirements into user stories, engineering tasks, and test cases — while also detecting ambiguities in the input and warning users to refine their requirements.

The application follows a modern microservices architecture with a Python/FastAPI backend and an Angular 17 frontend, containerised via Docker for consistent deployment.

Problem Statement

Translating raw BA requirements into well-structured Agile backlogs is time-consuming, error-prone, and heavily dependent on individual expertise. Requirements often contain ambiguous language, missing edge cases, and undefined scope. Manual creation of stories, tasks, and test cases from these requirements can take hours per feature. BA Bot automates this entire workflow.

Solution

  • Accepts requirements via text input, document upload (PDF/DOCX/TXT/CSV), or Jira references
  • Uses AI to generate stories with acceptance criteria and story-point estimates
  • Generates engineering tasks with assignee types and hour estimates
  • Detects requirement ambiguities and provides actionable suggestions
  • Generates deep, minimal test cases with step-by-step procedures
  • Supports full CRUD on generated work items (inline edit, delete, add new)
  • Persists analysis history in SQLite for browsing and rework
  • Exports results as JSON or Markdown

Data Flow

  1. User enters requirements text, uploads documents, and/or pastes Jira links
  2. Frontend sends POST /api/breakdown with the combined input
  3. Backend constructs a prompt and calls Anthropic API with tool-use to get structured JSON
  4. AI returns stories, tasks, ambiguity warnings in a single tool-use response
  5. Backend auto-saves the breakdown to SQLite and returns it with a history_id
  6. Frontend renders the interactive board with ambiguity warnings (if any)
  7. User can edit/add/delete items, generate test cases, save changes, or export

5c. AI Integration (anthropic_client.py)

The AI client uses Anthropic's tool-use feature to guarantee structured JSON output. This avoids fragile prompt-parsing and ensures every response matches the Pydantic schema exactly.

Breakdown Generation

  • Tool: output_breakdown — forces Claude to emit JSON matching the Breakdown schema
  • System prompt: Instructs the AI to act as a senior product manager / tech lead
  • Outputs: Summary, 3-8 stories with acceptance criteria, 4+ tasks with assignee types, and ambiguity warnings
  • Ambiguity detection: The prompt instructs the AI to flag vague terms, missing info, contradictions, multi-interpretable assumptions, and scope gaps

Test Case Generation

  • Tool: output_test_cases — forces Claude to emit structured test cases
  • System prompt: Senior QA engineer with strict rules:
    • Favour depth over breadth — as few test cases as possible
    • Merge scenarios into steps of existing cases when feasible
    • Only separate when preconditions or actors are fundamentally different
    • Cover happy paths first, then key negative/edge cases
  • Input: Full breakdown (stories + tasks + acceptance criteria) formatted as context

HTTP Client Configuration

Timeout: connect=10s, read=120s, write=30s, pool=10s
Limits:  max_connections=10, max_keepalive_connections=5
Model:   claude-sonnet-4-5 (configurable via ANTHROPIC_DEFAULT_MODEL env var)

8. User Flow & Screens

Screen 1: Requirements Input

  • Project name field (optional)
  • Requirements textarea (minimum 20 characters)
  • Document upload zone — drag-and-drop or browse for PDF/DOCX/TXT/CSV
  • Jira links textarea — paste URLs or issue keys
  • Extra context textarea — tech stack, constraints, etc.
  • Load Example button pre-fills a sample project
  • Generate Breakdown button triggers AI analysis
  • History panel below the form shows past analyses

Screen 2: Analysis Results

  • Result Header — project name and AI-generated summary
  • Ambiguity Banner (conditional) — amber warning panel listing each detected ambiguity with area, concern, and suggestion. Two actions: "Refine Input" (returns to form) or "Proceed Anyway"
  • Work Item Board — two-column grid:
    • Stories column — each card shows ID, title, description, acceptance criteria, story points. Expandable. Inline editable. Deletable with confirmation.
    • Tasks column — each card shows ID, title, description, assignee type, parent story, estimate hours. Same interaction patterns.
    • "Add Story" and "Add Task" buttons create new items in edit mode
  • Test Case Section — "Generate Test Cases" CTA button. Once generated, displays cards with ID, title, priority badge, story reference, preconditions, and step table (#, Action, Expected Result). Regenerate button available.

Toolbar Actions (when breakdown active)

  • Save — persists edits to the database
  • Export — JSON or Markdown download
  • History — toggles the history panel overlay
  • New — resets to the input form
